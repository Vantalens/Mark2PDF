# Quality Gates

版本：v0.1.0  
状态：生效  
最后更新：2026-04-26

## 默认验证

每轮开发默认运行：

```bash
npm test
```

当前测试必须覆盖：

- 核心转换 smoke test
- 转换快照测试
- 浏览器静态入口和自检页
- 本地安全 smoke test
- 资源预算 smoke test

## 开发完成门禁

- 当前任务验收通过。
- `DEVELOPMENT_TASKS.md` 已更新。
- 文档入口和专题文档没有冲突。
- 没有新增未批准依赖。
- 没有默认上传、遥测、远程转换或用户内容持久化。

## 插件门禁

插件能力进入主流程前必须满足：

- 有 manifest，声明格式、能力、体积、依赖、安全模式和加载方式。
- 不进入默认核心包，除非明确归类为 `format-basic`。
- 加载失败有用户可读错误和降级路径。
- 用户内容不会因为下载插件而被上传。
- `resource-budget-test` 能阻止重依赖进入核心路径。
- 至少有样例、快照或 smoke 覆盖。

## 基础包门禁

热门基础格式可以免下载内置，但必须满足：

- 属于高频转换路径。
- 不引入默认重依赖。
- 不需要默认网络访问。
- 不突破核心资源预算。
- 有样例、快照、warnings 和安全覆盖。

## 发布门禁

- P0 问题为 0。
- 核心流程可手工完成：上传、预览、转换、取消、下载。
- 安全策略和资源预算测试通过。
- CHANGELOG 已记录用户可见或开发者可见变化。
