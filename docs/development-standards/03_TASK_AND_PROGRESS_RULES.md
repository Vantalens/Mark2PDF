# Task And Progress Rules

版本：v0.1.0  
状态：生效  
最后更新：2026-04-26

## 任务粒度

每个任务应在 2 到 6 小时内能完成并验证。较大的目标必须拆成阶段任务，例如：

- 设计 manifest schema
- 实现插件加载器
- 添加按需下载 UI
- 增加资源预算测试
- 接入第一个 `format-plugin`

## 任务字段

新增任务至少包含：

- TASK ID：任务编号，例如 `P1-PLUGIN-01`
- 目标：要改变什么
- 输入：依赖哪些文档、模块、样例或设计
- 输出：会新增或修改哪些文件/能力
- 验收：用什么测试、smoke 或人工检查证明完成
- 回滚：失败时如何禁用或撤回

## Progress 规则

`DEVELOPMENT_TASKS.md` 是当前项目的 progress 主来源，必须保持：

- Done：已完成并验证的任务标记为 `[x]`
- In Progress：当前阶段状态写清下一步
- Next：未完成任务按优先级排序
- Blockers：阻塞项写进任务说明或新增风险条目

## 模块插件任务拆分

插件相关任务必须按以下顺序推进：

1. 文档与准入规则
2. manifest schema
3. 插件注册和能力发现
4. 按需加载和缓存策略
5. 安全边界和 local-only 校验
6. 资源预算测试
7. 首个插件样例
8. 用户界面提示和失败降级
